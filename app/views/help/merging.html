Merging Changes

This is a quick outline of how to merge changes. This may be applied to both upstream merge requests and also to and from feature branches. The primary thing to remember is the direction that you complete the merge in. In this example, we will merge a chance from the DEV branch to the MASTER branch;

For the impatient;

  git fetch origin
  git checkout -b dev origin/dev
  git checkout master
  git merge dev
  git push origin master

The complete explanation

You have a branch that you have been working on that is called dev. This is your feature branch that contains a set of changes. You want to commit it to the master branch so that people are able to grab it from there and work on it. This is also in preparation for it being moved in to the production branch once it has been tested and verified.

Step 1 - Fetch the origin source to make sure that you are in sync. 
  git fetch origin
Step 2 - Checkout the dev branch, based against origin/dev 
  git checkout -b dev origin/dev
Step 3 - Checkout the Master branch 
  git checkout master
Step 4 - Merge the Dev branch in to the currently checked out master

The direction in this step is important. You are asking to include changes from the dev branch in to the master branch. If you were currently working within the dev branch and you asked to merge in the master branch, then you would be taking changes FROM Master, not adding them to it.

  git merge dev
Step 5 - Push the changes back up to the remote server, to the branch Master 
  git push origin master
