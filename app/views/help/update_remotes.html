Update repository details

This outlines how to update your local repository to use the new gitlab remote sources. Currently we deal with only the unix options, but these same commands should be executable from the git-bash shell in windows.

For the impatient

cd  example_repo
git remote set-url origin git@gitlab.silverstripe.com.au:example_repo.git
git fetch origin

The longer explanation;

Each repo has remote sources that you commit your code back to. These are defined in the configuration of the actual repository. This means that each persons personal repository (local copy) may have totally different configuration All commands need to be run from within your local repository, so change in to the directory before executing any of the commands;

cd example_repo

In the above commands, I assume that you have the SSAU repository as the Origin remote. You can check what remotes you have configured within a repository with the following;

git remote -v

which should output somthing like this;

origin  git@cebada.silverstripe.com.au:example_repo.git(fetch)
origin  git@cebada.silverstripe.com.au:example_repo.git (push)

You can see that you have two items configured - A Fetch and a Push URL. For now, We are not worried about the difference, just that they are the same.

The format of the output is;

Remote Name, Remote URL

Now that we have that, we can replace the remote definition with a new URL using the 'remote set-url' option.

First, grab the new URL from the gitlab interface, then plug in the values;

git remote set-url origin git@gitlab.silverstripe.com.au:example_repo.git
Once you have configured the remote you can make sure that you can connect to the repo with a quick fetch 
git fetch origin

All being good, you will either get the latest updates, or you will be told that your repo is currently in sync.
